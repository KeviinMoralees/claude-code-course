7.1 — Que son los Subagentes

Un subagente es un asistente especializado que corre en su propia ventana de contexto, con un system prompt propio, acceso restringido a herramientas y permisos independientes. Cuando Claude encuentra una tarea que coincide con la descripcion de un subagente, delega el trabajo ahi y recibe el resultado de vuelta.

Los subagentes no pueden crear otros subagentes. Si necesitas anidamiento o delegacion en cadena, hazlo desde la conversacion principal. Cada subagente es un nodo terminal.

7.2 — Subagentes built-in

Claude Code incluye subagentes listos para usar que se activan automaticamente segun el contexto.

Subagente Modelo Herramientas Cuando Claude lo usa
Explore Haiku Solo lectura (sin Write/Edit) Busqueda y analisis de codigo sin modificar. Niveles: quick, medium, very thorough
Plan Hereda Solo lectura Investigacion de codebase durante Plan Mode antes de presentar un plan
general-purpose Hereda Todas Tareas complejas que requieren exploracion + modificacion + multiples pasos
statusline-setup Sonnet Limitadas Cuando corres /statusline para configurar la barra de estado
claude-code-guide Haiku Limitadas Cuando preguntas sobre funcionalidades de Claude Code

Cuando Claude necesita explorar el codebase, delega a Explore (Haiku, read-only). La exploracion ocurre en el contexto del subagente. Solo el resumen relevante sube a tu conversacion principal, ahorrando miles de tokens.

7.7 — Memoria persistente

Con el campo memory, el subagente tiene un directorio que sobrevive entre conversaciones. Va acumulando conocimiento institucional sobre el codebase, patrones, decisiones arquitectonicas.

Scope Ruta Cuando usarlo
user ~/.claude/agent-memory/<nombre>/ Conocimiento aplicable en todos los proyectos
project .claude/agent-memory/<nombre>/ Conocimiento del proyecto, compartible via git (recomendado)
local .claude/agent-memory-local/<nombre>/ Proyecto-especifico pero NO en version control

Cuando memory esta activo, el system prompt del subagente incluye instrucciones de lectura/escritura al directorio de memoria, y las primeras 200 lineas o 25KB de MEMORY.md. Read, Write y Edit se habilitan automaticamente para que pueda mantener sus propios archivos de memoria.

7.9 — Como invocar subagentes

Lenguaje natural (Claude decide si delega)

@-mention (garantiza que ese subagente corre)

Tu mensaje completo sigue yendo a Claude, que escribe el prompt del subagente. El @-mention controla cual subagente se invoca, no que instrucciones recibe.

--agent flag (toda la sesion como subagente)

Configuracion permanente por proyecto

Claude usa el campo description para decidir automaticamente cuando delegar. Para que un subagente se active con mas frecuencia, agrega frases como "Use proactively" en su descripcion.

7.10 — Foreground vs Background

Foreground Background
Bloquea conversacion Si No — continuas trabajando
Permission prompts Pasan al usuario Se aprueban antes de lanzar
AskUserQuestion Pasa al usuario Falla silenciosamente, el subagente continua
Como activar Default background: true en frontmatter o Ctrl+B

Si un subagente en background falla por permisos insuficientes, lanza uno en foreground con la misma tarea para reintentar con prompts interactivos. Para deshabilitar toda funcionalidad de background: CLAUDE_CODE_DISABLE_BACKGROUND_TASKS=1

7.11 — Patrones comunes

Aislar operaciones de alto volumen

El subagente corre miles de tokens de output de tests en su propio contexto. Solo el resumen sube al tuyo.

 
Run all the tests and show me the output

Todo el output de tests entra en tu contexto, consumiendo tokens rapidamente y comprimiendo tu historial.

Investigacion en paralelo

Cada subagente explora su area independientemente. Claude sintetiza los hallazgos. Funciona mejor cuando las areas de investigacion no dependen entre si.

Cuando los subagentes completan, sus resultados suben a tu conversacion principal. Si cada uno devuelve resultados muy detallados, puedes agotar el contexto rapidamente. Pide resumenes, no dumps completos.

Cadena de subagentes

Reanudar un subagente

Los subagentes reanudados retienen su historial completo: tool calls, resultados y razonamiento. Los transcripts se guardan en ~/.claude/projects/{project}/{sessionId}/subagents/.

7.12 — Cuando usar subagentes vs conversacion principal

7.13 — Agent Teams (experimental)

Es un concepto completamente distinto de los subagentes. Multiples sesiones de Claude Code coordinadas, cada una con su propia ventana de contexto completa, comunicandose directamente entre ellas via SendMessage.

Subagentes Agent Teams
Contexto Comparten la sesion principal Cada agente tiene su propio contexto completo
Comunicacion Solo reportan de vuelta al principal Se comunican directamente entre si
Interaccion Solo via conversacion principal Puedes hablar con cada agente individualmente
Estado Estable, disponible por defecto Experimental, desactivado por defecto

Agent Teams tiene limitaciones conocidas en: resumption de sesiones, coordinacion de tareas y comportamiento de shutdown. Usar en produccion con precaucion. Requiere Claude Code v2.1.32+.

Cuando usar Agent Teams vs Subagentes: si tu tarea requiere paralelismo sostenido o excede la ventana de contexto, Agent Teams da a cada trabajador su propio contexto independiente. Para la mayoria de casos, los subagentes regulares son suficientes y mas estables.

Buenas practicas de diseno

Un subagente, una responsabilidad. El debugger solo debuggea, el code-reviewer solo revisa. La descripcion es especifica e incluye "use proactively". Las herramientas son las minimas necesarias. Esta en version control para que el equipo lo use.

Un subagente "asistente general" con acceso a todas las herramientas y una descripcion vaga ("helps with coding tasks"). Claude nunca sabe cuando usarlo, y cuando lo usa puede hacer cambios destructivos donde no debia.
