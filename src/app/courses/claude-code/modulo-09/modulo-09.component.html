9.1 — GitHub Actions

Claude Code se integra con GitHub a través de la action oficial anthropics/claude-code-action@v1. Con un simple @claude en cualquier PR o issue, Claude puede analizar código, crear PRs, implementar features y corregir bugs — todo siguiendo los estándares del proyecto.

Setup rápido

La forma más fácil es ejecutar el comando desde el terminal de Claude Code:

Este comando guía la instalación de la GitHub App y los secrets necesarios. Requiere ser admin del repositorio. Para Bedrock/Vertex, hacer setup manual.

Setup manual (3 pasos)

  1. Instalar la GitHub App de Claude en el repositorio (permisos: Contents, Issues, Pull requests — lectura y escritura)
  2. Agregar ANTHROPIC_API_KEY a los secrets del repositorio
  3. Copiar el workflow desde examples/claude.yml a .github/workflows/

Si usas la versión beta, los nombres de parámetros cambiaron:

Enterprise: AWS Bedrock y Google Vertex AI

Para entornos enterprise con control de residencia de datos:

  • Usar siempre {{ apiKeyRef }} — nunca hardcodear la API key en el YAML
  • Configurar CLAUDE.md en el root del repo para que Claude respete los estándares del proyecto
  • Limitar permisos del job al mínimo necesario (contents: write, etc.)
  • Usar --max-turns para evitar iteraciones excesivas y costos inesperados
  • Configurar timeouts a nivel de job para evitar workflows que se queden colgados
  • Hardcodear la API key directamente en el workflow YAML — queda expuesta en git
  • No configurar --max-turns — Claude puede ejecutar muchas iteraciones costosas
  • Usar GITHUB_TOKEN (Actions user) en lugar de la GitHub App — los CI no se activan con commits del Actions user
  • Usar @beta en lugar de @v1 — la versión beta está deprecada

9.2 — Extensión VS Code

La extensión anthropic.claude-code proporciona una interfaz gráfica nativa integrada directamente en VS Code. La extensión incluye la CLI — comparten historial de conversaciones, settings y MCP servers.

Requisitos e instalación

  • VS Code 1.98.0 o superior
  • Cuenta de Anthropic (o proveedor de terceros configurado)
  • Instalar desde la marketplace: buscar "Claude Code" o presionar Cmd+Shift+X

Cómo abrir el panel

Modos de permiso

Modo Comportamiento Cuándo usar
Normal Claude pide permiso antes de cada acción Trabajo general, máximo control
Plan Claude describe el plan y espera aprobación. El plan se abre como markdown editable. Cambios complejos, multi-archivo
Auto-accept Claude hace edits sin pedir permiso Tareas rutinarias de confianza

@-mentions y referencias de archivos

Checkpoints — rewind del código

VS Code registra cada edición de Claude como checkpoint. Al pasar el cursor sobre cualquier mensaje, aparece el botón de rewind con tres opciones:

  • Fork conversation from here — nueva rama de conversación, código intacto
  • Rewind code to here — revierte archivos a este punto, conserva historial
  • Fork conversation and rewind code — nueva rama + código revertido

La extensión corre un servidor MCP local llamado ide que el CLI usa automáticamente. Expone dos herramientas al modelo: mcp__ide__getDiagnostics (errores/warnings del IDE) y mcp__ide__executeCode (ejecutar celdas Jupyter — siempre pide confirmación explícita). Este servidor no aparece en /mcp porque no necesita configuración.

9.3 — JetBrains IDEs

Claude Code se integra con los IDEs de JetBrains a través del plugin Claude Code [Beta] disponible en el JetBrains Marketplace. A diferencia de VS Code, no hay panel gráfico propio — la interacción ocurre desde el terminal integrado del IDE.

IDEs soportados

IntelliJ IDEA, PyCharm, WebStorm, Android Studio, PhpStorm, GoLand y la mayoría de IDEs JetBrains.

Instalación

  1. Buscar "Claude Code" en el JetBrains Marketplace e instalar
  2. Reiniciar el IDE completamente (puede requerir reiniciar varias veces)
  3. Ejecutar claude desde el terminal integrado del IDE

Features de integración

Shortcuts JetBrains

Acción Mac Windows/Linux
Abrir Claude Code Cmd+Esc Ctrl+Esc
Insertar referencia de archivo Cmd+Option+K Alt+Ctrl+K

Uso desde terminal externo

Configuración del plugin

Settings → Tools → Claude Code [Beta]:

  • Claude command: ruta personalizada si claude no está en PATH
  • Enable automatic updates: actualiza el plugin automáticamente al reiniciar
  • ESC key: si ESC no interrumpe operaciones, ir a Settings → Tools → Terminal y desvincular "Move focus to editor with Escape"

Remote Development: instalar el plugin en el host remoto, no en el cliente local. Ir a Settings → Plugin (Host).

WSL: configurar el Claude command como wsl -d Ubuntu -- bash -lic "claude" (reemplazar Ubuntu con el nombre de tu distribución).

JetBrains no tiene panel gráfico de Claude. Toda la interacción es a través del terminal integrado. Si buscas la experiencia visual (diffs inline, plan mode con markdown editable, checkpoints), usa VS Code.

9.4 — Scheduled Agents — Tareas Programadas

Claude Code puede ejecutar tareas de forma autónoma en un horario predefinido. La clave es ser explícito en el prompt sobre qué significa éxito y qué hacer con los resultados — la tarea corre sin supervisión humana.

Las cuatro opciones de scheduling

Opción Dónde corre Mejor para Configuración
Cloud scheduled tasks Infra de Anthropic Tareas que deben correr aunque la máquina esté apagada claude.ai/code
Desktop scheduled tasks Tu máquina (app desktop) Necesita acceso a archivos locales o cambios no comiteados App desktop → Schedule
GitHub Actions (schedule:) Runners de GitHub Tareas ligadas al repo, cron que vive junto al código .github/workflows/
/loop Sesión CLI activa Polling rápido mientras la sesión está abierta Comando en la sesión

GitHub Actions con cron — el más común

Referencia de sintaxis cron

Casos de uso típicos de tareas programadas

Ejemplo completo: audit de seguridad semanal

  • Ser explícito sobre qué significa éxito: "crea un issue con título X", "postea en canal Y"
  • Siempre incluir workflow_dispatch además del cron — permite pruebas manuales
  • Usar --max-turns para controlar costo — sin él, Claude puede iterar indefinidamente
  • Agregar --model claude-opus-4-6 para tareas complejas que requieren razonamiento profundo
  • Configurar timeouts a nivel de job: timeout-minutes: 30
  • Usar GitHub's concurrency controls para evitar runs paralelos del mismo job
  • Prompts ambiguos como "revisa el código" — Claude no puede pedir aclaraciones en modo automatización
  • No especificar dónde poner los resultados — Claude no sabrá qué hacer con sus hallazgos
  • Omitir --max-turns en tareas de larga duración — genera costos impredecibles
  • Correr audits de seguridad automáticos con acceso de escritura irrestricto sin revisión humana

9.5 — Git Workflows Avanzados

Claude Code tiene soporte nativo profundo para Git, desde commits y PRs hasta worktrees para trabajo paralelo aislado.

Comandos Git con Claude

Worktrees — trabajo paralelo sin conflictos

Los worktrees de Git crean directorios de trabajo separados con su propia rama, compartiendo el historial del repositorio. Permiten que múltiples sesiones de Claude trabajen simultáneamente sin interferirse.

Copiar archivos gitignored a worktrees

Los worktrees son checkouts frescos — no incluyen .env ni archivos locales. Usar .worktreeinclude para copiarlos automáticamente:

Solo se copian archivos que coincidan con el patrón y estén en .gitignore. Los archivos tracked nunca se duplican.

Cleanup de worktrees

  • Sin cambios: el worktree y su rama se eliminan automáticamente al salir
  • Con cambios: Claude pregunta si conservar o eliminar el worktree

Agregar .claude/worktrees/ al .gitignore del repo para evitar que el contenido de los worktrees aparezca como archivos sin trackear en el repositorio principal.
