5.1 — Que son los Hooks

Los hooks son comandos que se ejecutan automaticamente en puntos especificos del ciclo de vida de Claude Code. Proporcionan control deterministico — garantizan que ciertas acciones siempre ocurran, en vez de depender de que el LLM elija hacerlas.

4 tipos de hooks

Tipo Descripcion Caso de uso
command Ejecuta un comando shell El mas comun: scripts, formateo, logs
http POST de datos a un endpoint HTTP Audit logging, webhooks, integraciones
prompt Evaluacion single-turn con LLM (Haiku por default) Verificar calidad, validar completitud
agent Subagente multi-turn con acceso a herramientas Correr tests, verificaciones complejas

5.4 — Input y Output

Input (JSON via stdin)

Todos los eventos reciben campos base. Los tool events agregan datos de la herramienta:

Tool input por herramienta

Herramienta tool_input que recibe
Bash { "command": "npm test", "timeout": 120000 }
Write { "file_path": "/path", "content": "..." }

Glob { "pattern": "**/*.ts", "path": "/dir" }
Grep { "pattern": "regex", "path": "/path", "glob": "*.ts" }

Output (JSON via stdout)

Exit codes

Exit code Comportamiento
0 Exito. Se parsea JSON de stdout para decisiones
2 Error bloqueante. stderr se envia directamente a Claude
Otro Error no-bloqueante. stderr solo visible en modo verbose
Cuando multiples hooks matchean el mismo evento, la decision mas restrictiva gana: deny > defer > ask > allow El additionalContext inyectado al contexto esta limitado a 10,000 caracteres. Lo que exceda se guarda en archivo con un preview y ruta.

5.5 — PreToolUse (el mas poderoso)

Intercepta antes de que una herramienta se ejecute. Puede permitir, denegar o pedir confirmacion. Es el hook mas usado para seguridad y validacion.

Decisiones posibles

Decision Efecto
allow Salta el prompt de permiso (deny rules siguen aplicando)
deny Cancela la herramienta, envia razon a Claude
ask Muestra el prompt de permiso al usuario
defer Disponible en modo no-interactivo (-p)

5.6 — PostToolUse (despues de ejecutar)

Se ejecuta despues de que una herramienta termina exitosamente. No puede deshacer la accion — sirve para formatear, loguear, notificar e inyectar contexto.

5.8 — Seguridad y limitaciones

Modelo de seguridad

Timeouts

Tipo Default Configurable?
command 600 segundos (10 min) Si, via timeout
prompt 30 segundos Si
agent 60 segundos, max 50 turns Si

Gotchas comunes

Problema Solucion
echo en shell profile corrompe JSON output Envolver en if [[ $- == *i* ]]; then
Script no ejecutable chmod +x script.sh
Matchers son case-sensitive Bash no es bash
PermissionRequest no dispara en -p Usar PreToolUse en modo headless
Stop hook en cada respuesta, no solo al final Verificar stop_hook_active para evitar loops

Variables de entorno disponibles

Variable Disponible en
$CLAUDE_PROJECT_DIR Todos los hooks
$CLAUDE_ENV_FILE SessionStart, CwdChanged, FileChanged
$CLAUDE_CODE_REMOTE Todos ("true" en entornos web remotos)

Recetas practicas completas
