9.1 — GitHub Actions

Claude Code se integra con GitHub a través de la action oficial anthropics/claude-code-action@v1. Con un simple @claude en cualquier PR o issue, Claude puede analizar código, crear PRs, implementar features y corregir bugs — todo siguiendo los estándares del proyecto.

Setup rápido

La forma más fácil es ejecutar el comando desde el terminal de Claude Code:

/install-github-app

Este comando guía la instalación de la GitHub App y los secrets necesarios. Requiere ser admin del repositorio. Para Bedrock/Vertex, hacer setup manual.

Setup manual (3 pasos)

  1. Instalar la GitHub App de Claude en el repositorio (permisos: Contents, Issues, Pull requests — lectura y escritura)
  2. Agregar ANTHROPIC_API_KEY a los secrets del repositorio
  3. Copiar el workflow desde examples/claude.yml a .github/workflows/

Dos modos de operación (auto-detectados en v1)

Modo interactivo

Responde a menciones @claude en comentarios de PRs e issues. No requiere prompt.

Modo automatización

Se ejecuta con un prompt fijo cuando ocurre un evento (PR abierto, cron, push). No necesita trigger humano.

Triggers disponibles

on:
  issue_comment:           # comentarios en issues y PRs
    types: [created]
  pull_request_review_comment:
    types: [created]
  issues:                   # nuevo issue abierto/asignado
    types: [opened, assigned]
  pull_request:             # PR abierto o actualizado
    types: [opened, synchronize]
  schedule:                 # cron programado
    - cron: "0 9 * * 1-5"
  push:                     # push a rama específica
    branches: [main]

Workflow básico — responde a @claude

name: Claude Code
on:
  issue_comment:
    types: [created]
  pull_request_review_comment:
    types: [created]

jobs:
  claude:
    runs-on: ubuntu-latest
    steps:
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}

Workflow de code review automático

name: Automated PR Review
on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          prompt: "Review this PR for code quality, security issues,
            and adherence to project standards. Post inline review comments."
          claude_args: "--max-turns 5 --model claude-sonnet-4-6"

Enterprise: AWS Bedrock y Google Vertex AI

Para entornos enterprise con control de residencia de datos:

# AWS Bedrock con OIDC (sin credenciales estáticas)
- name: Configure AWS Credentials
  uses: aws-actions/configure-aws-credentials@v4
  with:
    role-to-assume: ${{ secrets.AWS_ROLE_TO_ASSUME }}
    aws-region: us-west-2

- uses: anthropics/claude-code-action@v1
  with:
    github_token: ${{ steps.app-token.outputs.token }}
    use_bedrock: "true"
    claude_args: '--model us.anthropic.claude-sonnet-4-6 --max-turns 10'
✓ Buenas prácticas GitHub Actions
  • Usar siempre ${{ secrets.ANTHROPIC_API_KEY }} — nunca hardcodear la API key en el YAML
  • Configurar CLAUDE.md en el root del repo para que Claude respete los estándares del proyecto
  • Limitar permisos del job al mínimo necesario (contents: write, etc.)
  • Usar --max-turns para evitar iteraciones excesivas y costos inesperados
  • Configurar timeouts a nivel de job para evitar workflows que se queden colgados
✗ Errores comunes
  • Hardcodear la API key directamente en el workflow YAML — queda expuesta en git
  • No configurar --max-turns — Claude puede ejecutar muchas iteraciones costosas
  • Usar GITHUB_TOKEN (Actions user) en lugar de la GitHub App — los CI no se activan con commits del Actions user
  • Usar @beta en lugar de @v1 — la versión beta está deprecada

9.2 — Extensión VS Code

La extensión anthropic.claude-code proporciona una interfaz gráfica nativa integrada directamente en VS Code. La extensión incluye la CLI — comparten historial de conversaciones, settings y MCP servers.

Requisitos e instalación

  • VS Code 1.98.0 o superior
  • Cuenta de Anthropic (o proveedor de terceros configurado)
  • Instalar desde la marketplace: buscar "Claude Code" o presionar Cmd+Shift+X

Cómo abrir el panel

Icono ✦ Editor Toolbar

Esquina superior derecha del editor. Sólo aparece cuando hay un archivo abierto. Forma más rápida.

Activity Bar

Barra lateral izquierda — siempre visible. Abre lista de sesiones. Click en sesión = tab completo.

Status Bar

✱ Claude Code en la esquina inferior derecha. Funciona aunque no haya archivos abiertos.

Command Palette

Cmd+Shift+P → buscar 'Claude Code'. Opciones: abrir en tab, sidebar, nueva ventana.

Modos de permiso

ModoComportamientoCuándo usar
NormalClaude pide permiso antes de cada acciónTrabajo general, máximo control
PlanClaude describe el plan y espera aprobación. El plan se abre como markdown editable.Cambios complejos, multi-archivo
Auto-acceptClaude hace edits sin pedir permisoTareas rutinarias de confianza

Checkpoints — rewind del código

VS Code registra cada edición de Claude como checkpoint. Al pasar el cursor sobre cualquier mensaje, aparece el botón de rewind con tres opciones:

  • Fork conversation from here — nueva rama de conversación, código intacto
  • Rewind code to here — revierte archivos a este punto, conserva historial
  • Fork conversation and rewind code — nueva rama + código revertido

Abrir Claude desde scripts externos

# URI handler — abre nueva tab en VS Code
open "vscode://anthropic.claude-code/open"

# Con prompt pre-cargado
open "vscode://anthropic.claude-code/open?prompt=review%20my%20changes"

# Retomando sesión específica
open "vscode://anthropic.claude-code/open?session=abc123"

ℹ MCP server IDE interno

La extensión corre un servidor MCP local llamado ide que el CLI usa automáticamente. Expone dos herramientas al modelo: mcp__ide__getDiagnostics (errores/warnings del IDE) y mcp__ide__executeCode (ejecutar celdas Jupyter — siempre pide confirmación explícita). Este servidor no aparece en /mcp porque no necesita configuración.

9.3 — JetBrains IDEs

Claude Code se integra con los IDEs de JetBrains a través del plugin Claude Code [Beta] disponible en el JetBrains Marketplace. A diferencia de VS Code, no hay panel gráfico propio — la interacción ocurre desde el terminal integrado del IDE.

IDEs soportados

IntelliJ IDEA, PyCharm, WebStorm, Android Studio, PhpStorm, GoLand y la mayoría de IDEs JetBrains.

Instalación

  1. Buscar "Claude Code" en el JetBrains Marketplace e instalar
  2. Reiniciar el IDE completamente (puede requerir reiniciar varias veces)
  3. Ejecutar claude desde el terminal integrado del IDE

Features de integración

Diff viewer nativo

Los cambios de Claude se muestran en el visor de diffs del IDE en lugar del terminal.

Contexto de selección

La selección activa del editor se comparte automáticamente con Claude.

Diagnósticos automáticos

Errores de lint y sintaxis del IDE se comparten con Claude mientras trabajas.

File reference shortcuts

Insertar referencias @File#L1-99 directamente desde el editor.

Shortcuts JetBrains

AcciónMacWindows/Linux
Abrir Claude CodeCmd+EscCtrl+Esc
Insertar referencia de archivoCmd+Option+KAlt+Ctrl+K

Uso desde terminal externo

# Conectar Claude Code externo al IDE JetBrains activo
claude
/ide

# Asegurarse de iniciar Claude desde el mismo directorio que el proyecto
cd /path/to/project && claude

Configuración del plugin

Settings → Tools → Claude Code [Beta]:

  • Claude command: ruta personalizada si claude no está en PATH
  • Enable automatic updates: actualiza el plugin automáticamente al reiniciar
  • ESC key: si ESC no interrumpe operaciones, ir a Settings → Tools → Terminal y desvincular "Move focus to editor with Escape"
⚠ Casos especiales

Remote Development: instalar el plugin en el host remoto, no en el cliente local. Ir a Settings → Plugin (Host).

WSL: configurar el Claude command como wsl -d Ubuntu -- bash -lic "claude" (reemplazar Ubuntu con el nombre de tu distribución).

✗ Diferencia clave vs VS Code

JetBrains no tiene panel gráfico de Claude. Toda la interacción es a través del terminal integrado. Si buscas la experiencia visual (diffs inline, plan mode con markdown editable, checkpoints), usa VS Code.

9.4 — Scheduled Agents — Tareas Programadas

Claude Code puede ejecutar tareas de forma autónoma en un horario predefinido. La clave es ser explícito en el prompt sobre qué significa éxito y qué hacer con los resultados — la tarea corre sin supervisión humana.

Las cuatro opciones de scheduling

OpciónDónde correMejor paraConfiguración
Cloud scheduled tasksInfra de AnthropicTareas que deben correr aunque la máquina esté apagadaclaude.ai/code
Desktop scheduled tasksTu máquina (app desktop)Necesita acceso a archivos locales o cambios no comiteadosApp desktop → Schedule
GitHub Actions (schedule:)Runners de GitHubTareas ligadas al repo, cron que vive junto al código.github/workflows/
/loopSesión CLI activaPolling rápido mientras la sesión está abiertaComando en la sesión

GitHub Actions con cron — el más común

name: Daily PR Review
on:
  schedule:
    - cron: "0 9 * * 1-5"   # 9am lunes a viernes UTC

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          prompt: "Review all open PRs labeled 'needs-review'.
            Post inline comments for any issues found.
            Post a summary in the PR description when done."
          claude_args: "--max-turns 15 --model claude-sonnet-4-6"

Referencia de sintaxis cron

# Formato: minuto hora día-del-mes mes día-de-la-semana
"0 9 * * *"        # todos los días a las 9am UTC
"0 9 * * 1-5"      # lunes a viernes 9am UTC
"0 */6 * * *"      # cada 6 horas
"0 9 1 * *"        # primer día de cada mes 9am
"30 14 * * 5"      # viernes 2:30pm UTC

# Nota: GitHub Actions cron usa UTC siempre

Casos de uso típicos de tareas programadas

Daily standup report

Cada mañana: resume commits de ayer, issues abiertos, PRs bloqueados. Postea en Slack o crea un issue.

Dependency audit semanal

Cada lunes: revisa si hay vulnerabilidades en dependencias y crea PR con actualizaciones.

Code review automático

En cada PR abierto: Claude revisa calidad, seguridad y adherencia a estándares.

Stale PR cleanup

Semanal: identifica PRs sin actividad por más de 2 semanas y agrega etiqueta o comenta.

Ejemplo completo: audit de seguridad semanal

name: Weekly Security Audit
on:
  schedule:
    - cron: "0 8 * * 1"    # lunes 8am UTC
  workflow_dispatch:              # también permite ejecución manual

permissions:
  contents: write
  issues: write
  pull-requests: write

jobs:
  security-audit:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          prompt: "Perform a security audit of this codebase:
            1. Check for hardcoded secrets or credentials
            2. Review authentication and authorization logic
            3. Check for common vulnerabilities (SQL injection, XSS, etc.)
            4. Review dependency versions for known CVEs
            Create a GitHub issue titled 'Weekly Security Report - [date]'
            with your findings categorized by severity (Critical/High/Medium/Low).
            If you find critical issues, label the issue as 'critical'."
          claude_args: "--max-turns 20 --model claude-opus-4-6"
✓ Buenas prácticas para tareas programadas
  • Ser explícito sobre qué significa éxito: "crea un issue con título X", "postea en canal Y"
  • Siempre incluir workflow_dispatch además del cron — permite pruebas manuales
  • Usar --max-turns para controlar costo — sin él, Claude puede iterar indefinidamente
  • Agregar --model claude-opus-4-6 para tareas complejas que requieren razonamiento profundo
  • Configurar timeouts a nivel de job: timeout-minutes: 30
  • Usar GitHub's concurrency controls para evitar runs paralelos del mismo job
✗ Anti-patrones en scheduled agents
  • Prompts ambiguos como "revisa el código" — Claude no puede pedir aclaraciones en modo automatización
  • No especificar dónde poner los resultados — Claude no sabrá qué hacer con sus hallazgos
  • Omitir --max-turns en tareas de larga duración — genera costos impredecibles
  • Correr audits de seguridad automáticos con acceso de escritura irrestricto sin revisión humana

9.5 — Git Workflows Avanzados

Claude Code tiene soporte nativo profundo para Git, desde commits y PRs hasta worktrees para trabajo paralelo aislado.

Comandos Git con Claude

# Claude puede hacer todo el flujo git por ti
commit my changes with a descriptive message
create a pr for this feature with a detailed description
summarize the changes I've made to the auth module

# Resume sesión ligada a un PR específico
claude --from-pr 123

# Setup GitHub App (desde dentro de Claude)
/install-github-app

Worktrees — trabajo paralelo sin conflictos

Los worktrees de Git crean directorios de trabajo separados con su propia rama, compartiendo el historial del repositorio. Permiten que múltiples sesiones de Claude trabajen simultáneamente sin interferirse. 

# Crear worktree con nombre específico
claude --worktree feature-auth
# Crea .claude/worktrees/feature-auth/ con rama worktree-feature-auth

# Worktree con nombre auto-generado
claude --worktree
# Genera nombre como "bright-running-fox"

# Dos sesiones paralelas sin conflictos
claude --worktree feature-payments   # terminal 1
claude --worktree bugfix-login       # terminal 2

Copiar archivos gitignored a worktrees

Los worktrees son checkouts frescos — no incluyen .env ni archivos locales. Usar .worktreeinclude para copiarlos automáticamente: 

# .worktreeinclude en la raíz del repo
.env
.env.local
config/secrets.json
.credentials

Solo se copian archivos que coincidan con el patrón y estén en .gitignore. Los archivos tracked nunca se duplican.

Cleanup de worktrees

  • Sin cambios: el worktree y su rama se eliminan automáticamente al salir
  • Con cambios: Claude pregunta si conservar o eliminar el worktree
# Gestión manual de worktrees
git worktree list
git worktree remove .claude/worktrees/feature-auth

# Re-sincronizar rama base si cambió el default branch en GitHub
git remote set-head origin -a
ℹ .gitignore recomendado

Agregar .claude/worktrees/ al .gitignore del repo para evitar que el contenido de los worktrees aparezca como archivos sin trackear en el repositorio principal.

Resumir conversaciones

# Retomar la conversación más reciente del directorio actual
claude --continue

# Abrir picker interactivo de conversaciones
claude --resume

# Retomar por nombre
claude --resume auth-refactor

# Nombrar sesión al iniciar
claude -n auth-refactor

# Retomar sesión ligada a un PR
claude --from-pr 456
