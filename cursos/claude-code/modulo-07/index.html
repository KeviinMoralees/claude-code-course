Subagentes y Agent Teams

Asistentes especializados con su propia ventana de contexto, herramientas restringidas y memoria persistente

7.1 — Que son los Subagentes

Un subagente es un asistente especializado que corre en su propia ventana de contexto, con un system prompt propio, acceso restringido a herramientas y permisos independientes. Cuando Claude encuentra una tarea que coincide con la descripcion de un subagente, delega el trabajo ahi y recibe el resultado de vuelta.

Preservar contexto

Operaciones verbosas (tests, logs, exploracion masiva) se quedan en el contexto del subagente, no en el tuyo.

Aplicar restricciones

Limita que herramientas puede usar para una tarea. Un revisor que solo lee, no puede escribir accidentalmente.

Especializacion

System prompts enfocados en un dominio: code review, debugging, analisis de datos, queries de DB.

Control de costos

Rutea tareas de exploracion a Haiku en vez de Opus. El subagente Explore ya hace esto por defecto.

Limitacion importante

Los subagentes no pueden crear otros subagentes. Si necesitas anidamiento o delegacion en cadena, hazlo desde la conversacion principal. Cada subagente es un nodo terminal.

7.2 — Subagentes built-in

Claude Code incluye subagentes listos para usar que se activan automaticamente segun el contexto.

SubagenteModeloHerramientasCuando Claude lo usa
ExploreHaikuSolo lectura (sin Write/Edit)Busqueda y analisis de codigo sin modificar. Niveles: quick, medium, very thorough
PlanHeredaSolo lecturaInvestigacion de codebase durante Plan Mode antes de presentar un plan

Explore y la ventana de contexto

Cuando Claude necesita explorar el codebase, delega a Explore (Haiku, read-only). La exploracion ocurre en el contexto del subagente. Solo el resumen relevante sube a tu conversacion principal, ahorrando miles de tokens.

7.3 — Crear subagentes

Via /agents (recomendado)

El comando /agents abre una interfaz interactiva para crear, editar, eliminar y ver todos los subagentes disponibles. Es la forma recomendada. 

/agents  → "Create new agent" → Personal/Project → "Generate with Claude"

# Desde CLI sin sesion interactiva:
claude agents  → lista todos los agentes agrupados por fuente

Manual — archivo Markdown con frontmatter

# Personal (todos los proyectos)
~/.claude/agents/code-reviewer.md

# Proyecto (compartible via git)
.claude/agents/code-reviewer.md

Estructura basica de un subagente

---
name: code-reviewer
description: Expert code reviewer. Proactively reviews code for quality,
  security, and maintainability. Use immediately after writing or modifying code.
tools: Read, Grep, Glob, Bash
model: inherit
---

You are a senior code reviewer ensuring high standards.

When invoked:
1. Run git diff to see recent changes
2. Focus on modified files
3. Begin review immediately

Review checklist:
- Code clarity and readability
- No exposed secrets or API keys
- Input validation implemented
- Proper error handling and test coverage

Output: critical issues, warnings, suggestions (ordered by priority)
Los subagentes se cargan al inicio de sesion

Si creas un subagente manualmente agregando un archivo, reinicia la sesion o usa /agents para cargarlo de inmediato sin reiniciar.

Util para testing rapido o scripts de automatizacion. No se guarda en disco.

7.6 — Control de herramientas y permisos

Allowlist vs Denylist

# Allowlist — SOLO estas herramientas
tools: Read, Grep, Glob, Bash

# Denylist — todo excepto estas
disallowedTools: Write, Edit

# Si ambas estan definidas:
# primero se aplica disallowedTools, luego se resuelve tools contra lo que queda
# un tool en ambas listas queda removido

Restringir que subagentes puede crear

Solo aplica cuando el agente corre como sesion principal con --agent. Los subagentes regulares no pueden crear otros subagentes. 

# Allowlist de subagentes que puede crear
tools: Agent(worker, researcher), Read, Bash

# Sin restriccion de subagentes
tools: Agent, Read, Bash

# No puede crear ningun subagente (omitir Agent del tools)
tools: Read, Bash

MCP Servers con scope al subagente

---
name: browser-tester
description: Tests features in a real browser using Playwright
mcpServers:
  # Definicion inline: scoped solo a este subagente
  - playwright:
      type: stdio
      command: npx
      args: ["-y", "@playwright/mcp@latest"]
  # Referencia a servidor ya configurado en la sesion
  - github
---

Use the Playwright tools to navigate, screenshot, and interact with pages.

Los servidores inline se conectan al inicio del subagente y se desconectan al terminar. Si quieres mantener un MCP server fuera de la conversacion principal (para no consumir tokens con sus tool descriptions), definelo inline aqui.

Deshabilitar subagentes especificos

// settings.json
{
  "permissions": {
    "deny": ["Agent(Explore)", "Agent(my-custom-agent)"]
  }
}

# O via CLI flag:
claude --disallowedTools "Agent(Explore)"

7.7 — Memoria persistente

Con el campo memory, el subagente tiene un directorio que sobrevive entre conversaciones. Va acumulando conocimiento institucional sobre el codebase, patrones, decisiones arquitectonicas.

ScopeRutaCuando usarlo
user~/.claude/agent-memory/<nombre>/Conocimiento aplicable en todos los proyectos
project.claude/agent-memory/<nombre>/Conocimiento del proyecto, compartible via git (recomendado)
local.claude/agent-memory-local/<nombre>/Proyecto-especifico pero NO en version control
---
name: code-reviewer
description: Reviews code and accumulates project knowledge over time
memory: project
---

You are a code reviewer. As you work, update your agent memory with:
- Codepaths and their purpose
- Recurring issues and patterns you observe
- Architectural decisions and conventions

Before reviewing, check your memory for patterns you've seen before.
After finishing, save what you learned to your memory.
Como funciona internamente

Cuando memory esta activo, el system prompt del subagente incluye instrucciones de lectura/escritura al directorio de memoria, y las primeras 200 lineas o 25KB de MEMORY.md. Read, Write y Edit se habilitan automaticamente para que pueda mantener sus propios archivos de memoria.

7.8 — Hooks en subagentes

Hooks en el frontmatter del subagente

Solo se ejecutan mientras ese subagente esta activo.

---
name: db-reader
description: Execute read-only database queries
tools: Bash
hooks:
  PreToolUse:
    - matcher: "Bash"
      hooks:
        - type: command
          command: "./scripts/validate-readonly-query.sh"
  PostToolUse:
    - matcher: "Edit|Write"
      hooks:
        - type: command
          command: "./scripts/run-linter.sh"
---

Script de validacion de solo lectura:

#!/bin/bash
# ./scripts/validate-readonly-query.sh

INPUT=$(cat)
COMMAND=$(echo "$INPUT" | jq -r '.tool_input.command // empty')

# Bloquear operaciones de escritura SQL
if echo "$COMMAND" | grep -iE '(INSERT|UPDATE|DELETE|DROP|CREATE|ALTER|TRUNCATE)' > /dev/null; then
  echo "Blocked: Only SELECT queries are allowed" >&2
  exit 2
fi

exit 0

Hooks en settings.json para ciclo de vida del subagente

// settings.json
{
  "hooks": {
    "SubagentStart": [
      {
        "matcher": "db-agent",
        "hooks": [
          { "type": "command", "command": "./scripts/setup-db-connection.sh" }
        ]
      }
    ],
    "SubagentStop": [
      {
        "hooks": [
          { "type": "command", "command": "./scripts/cleanup.sh" }
        ]
      }
    ]
  }
}
EventoMatcher inputCuando se dispara
SubagentStartNombre del tipo de agenteCuando un subagente comienza ejecucion
SubagentStopNombre del tipo de agenteCuando un subagente completa su trabajo

7.9 — Como invocar subagentes

Lenguaje natural (Claude decide si delega)

Use the code-reviewer subagent to look at my recent changes
Have the debugger subagent analyze this error

@-mention (garantiza que ese subagente corre)

@"code-reviewer (agent)" review the auth module
@"debugger (agent)" fix the failing tests in src/api/

Tu mensaje completo sigue yendo a Claude, que escribe el prompt del subagente. El @-mention controla cual subagente se invoca, no que instrucciones recibe.

--agent flag (toda la sesion como subagente)

# La sesion completa corre con ese system prompt
claude --agent code-reviewer

# Plugin subagent
claude --agent my-plugin:specialized-agent

Configuracion permanente por proyecto

// .claude/settings.json
{
  "agent": "code-reviewer"
}
Descripcion como trigger

Claude usa el campo description para decidir automaticamente cuando delegar. Para que un subagente se active con mas frecuencia, agrega frases como "Use proactively" en su descripcion.

7.10 — Foreground vs Background

ForegroundBackground
Bloquea conversacionSiNo — continuas trabajando

Como activarDefaultbackground: true en frontmatter o Ctrl+B
Background y permisos

Si un subagente en background falla por permisos insuficientes, lanza uno en foreground con la misma tarea para reintentar con prompts interactivos. Para deshabilitar toda funcionalidad de background: CLAUDE_CODE_DISABLE_BACKGROUND_TASKS=1

7.11 — Patrones comunes

Aislar operaciones de alto volumen

Bien — el output verboso queda fuera de tu contexto
Research the authentication, database, and API modules
in parallel using separate subagents

El subagente corre miles de tokens de output de tests en su propio contexto. Solo el resumen sube al tuyo.

Mal — correr tests en la conversacion principal
Run all the tests and show me the output

Todo el output de tests entra en tu contexto, consumiendo tokens rapidamente y comprimiendo tu historial.

Investigacion en paralelo

Research the authentication, database, and API modules
in parallel using separate subagents

Cada subagente explora su area independientemente. Claude sintetiza los hallazgos. Funciona mejor cuando las areas de investigacion no dependen entre si.

Cuidado con la sintesis

Cuando los subagentes completan, sus resultados suben a tu conversacion principal. Si cada uno devuelve resultados muy detallados, puedes agotar el contexto rapidamente. Pide resumenes, no dumps completos.

Cadena de subagentes

Use the code-reviewer to find performance issues,
then use the optimizer subagent to fix them

Reanudar un subagente

Use the code-reviewer subagent to review the authentication module
[Subagente completa]

Continue that code review and now analyze the authorization logic
[Claude reanuda el subagente con historial completo]

Los subagentes reanudados retienen su historial completo: tool calls, resultados y razonamiento. Los transcripts se guardan en ~/.claude/projects/{project}/{sessionId}/subagents/.

7.12 — Cuando usar subagentes vs conversacion principal

Usa la conversacion principal cuando...

Necesitas ida y vuelta iterativa. Multiples fases comparten contexto significativo. Es un cambio rapido y puntual. La latencia importa (los subagentes arrancan desde cero).

Usa subagentes cuando...

La tarea produce output verboso que no quieres en tu contexto. Quieres restricciones especificas de herramientas o permisos. El trabajo es autocontenido y puede devolver un resumen. Quieres paralelismo real.

Considera Skills cuando...

Quieres prompts o workflows reutilizables que corran en el contexto principal (no aislado). Las Skills son instrucciones; los subagentes son contextos separados.

Usa /btw cuando...

Tienes una pregunta rapida sobre algo que ya esta en tu conversacion. Ve el contexto completo pero sin acceso a herramientas. La respuesta se descarta del historial.

7.13 — Agent Teams (experimental)

Es un concepto completamente distinto de los subagentes. Multiples sesiones de Claude Code coordinadas, cada una con su propia ventana de contexto completa, comunicandose directamente entre ellas via SendMessage.

SubagentesAgent Teams
ContextoComparten la sesion principalCada agente tiene su propio contexto completo
ComunicacionSolo reportan de vuelta al principalSe comunican directamente entre si
InteraccionSolo via conversacion principalPuedes hablar con cada agente individualmente
EstadoEstable, disponible por defectoExperimental, desactivado por defecto
// settings.json — habilitar Agent Teams
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
Estado actual

Agent Teams tiene limitaciones conocidas en: resumption de sesiones, coordinacion de tareas y comportamiento de shutdown. Usar en produccion con precaucion. Requiere Claude Code v2.1.32+.

Cuando usar Agent Teams vs Subagentes: si tu tarea requiere paralelismo sostenido o excede la ventana de contexto, Agent Teams da a cada trabajador su propio contexto independiente. Para la mayoria de casos, los subagentes regulares son suficientes y mas estables.

7.14 — Ejemplos completos

Debugger con acceso completo

---
name: debugger
description: Debugging specialist for errors, test failures, and unexpected
  behavior. Use proactively when encountering any issues.
tools: Read, Edit, Bash, Grep, Glob
---

You are an expert debugger specializing in root cause analysis.

When invoked:
1. Capture error message and stack trace
2. Identify reproduction steps
3. Isolate the failure location
4. Implement minimal fix
5. Verify solution works

For each issue, provide:
- Root cause explanation with evidence
- Specific code fix
- Prevention recommendations

Data scientist especializado

---
name: data-scientist
description: Data analysis expert for SQL queries, BigQuery operations, and data
  insights. Use proactively for data analysis tasks and queries.
tools: Bash, Read, Write
model: sonnet
---

You are a data scientist specializing in SQL and BigQuery analysis.

When invoked:
1. Understand the analysis requirement
2. Write efficient SQL queries
3. Analyze and summarize results
4. Provide data-driven recommendations

Key practices:
- Write optimized queries with proper filters
- Include comments explaining complex logic
- Highlight key findings and suggest next steps

Buenas practicas de diseno

Subagente bien disenado

Un subagente, una responsabilidad. El debugger solo debuggea, el code-reviewer solo revisa. La descripcion es especifica e incluye "use proactively". Las herramientas son las minimas necesarias. Esta en version control para que el equipo lo use.

Subagente mal disenado

Un subagente "asistente general" con acceso a todas las herramientas y una descripcion vaga ("helps with coding tasks"). Claude nunca sabe cuando usarlo, y cuando lo usa puede hacer cambios destructivos donde no debia.
